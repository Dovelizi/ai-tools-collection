# STAR-Resume-Pro

## Overview

STAR-Resume-Pro 是一个专业简历生成技能，核心特色：

1. **STAR 法则驱动**：所有工作/项目描述严格遵循 Situation - Task - Action - Result 结构
2. **量化收益与成本**：每条成果必须包含可衡量的数据指标（效率提升%、成本节省、时间缩短等）
3. **现代简历设计**：采用 2025-2026 年最流行的双栏侧边栏布局，柔和配色，清晰视觉层次
4. **ATS 友好**：结构化 HTML 语义标签，确保机器和人类都能高效解析

## 执行工作流

### 阶段零：信息收集

开始时向用户说明：

"您好！我是 STAR-Resume-Pro 简历助手。我将帮您打造一份以 STAR 法则和量化数据为核心的专业简历。请回答我的问题，或直接提供已有简历/PDF 文件，我会自动提取并优化。"

**检查已有简历**：静默扫描工作目录中的 PDF/DOCX 文件。若存在，使用对应 skill 读取并提取内容，向用户确认后进入优化流程。

### 阶段一：逐模块收集信息

按以下顺序一次性询问一个模块（避免信息过载），获取完整信息后立即应用 STAR 优化：

#### 模块 1：个人信息
- 姓名（必填）
- 目标岗位/求职意向（必填）
- 手机号（必填）
- 邮箱（必填）
- 个人主页/GitHub/LinkedIn（选填）
- 所在城市（选填）

#### 模块 2：专业摘要
收集用户的核心竞争力，撰写 2-3 句话的专业摘要（Professional Summary），要求：
- 突出年限 + 领域 + 核心技能 + 最亮眼量化成果
- 示例："5年全栈开发经验，专注高并发分布式系统，主导3个核心项目从0到1落地，累计为公司节省运营成本200万/年，系统可用性提升至 99.99%。"

#### 模块 3：技能清单
收集技术栈/核心技能，按以下维度分类：
- 编程语言 / 框架
- 工具与平台
- 软技能 / 语言能力
- 证书与资质

#### 模块 4：工作经历
逐段收集，每段包括：
- 公司名称 + 岗位 + 起止时间
- 用户自然语言描述的工作内容（至少 2-3 项）

**STAR 优化规则**（参阅 references/star_guide.md）：
将每条工作内容按以下公式重写：

```
[强力动词] + [做了什么] + 通过/使用 [方法/工具] + 以实现 [目标]，最终 [量化结果] + 影响范围
```

每条描述必须：
1. 以强力动词开头（主导/设计/优化/构建/推动/重构/搭建）
2. 包含具体技术手段或方法论
3. 以量化结果收尾（参阅 references/quantify_guide.md）
4. 控制在 1-2 行以内（40-80 字）

#### 模块 5：项目经历
逐段收集，每段包括：
- 项目名称 + 角色 + 起止时间
- 项目背景（1 句话）
- 工作内容与成果

同样应用 STAR 优化规则。

#### 模块 6：教育背景
- 学校 + 专业 + 学历 + 起止时间
- GPA / 排名（选填）
- 荣誉 / 奖学金（选填）

### 阶段二：STAR 深度优化

收集完所有信息后，对每条经历描述执行以下检查：

**STAR 完整性检查清单**：
- S（情境）：是否有 3-6 词的背景交代？
- T（任务）：是否明确了目标/挑战？
- A（行动）：是否说明了具体做法和使用的工具？
- R（结果）：是否有量化数据？

**量化维度检查**（必须至少命中 1 个）：
- **效率**：时间缩短 X%、节省 X 小时/周
- **成本**：节省 X 万/年、降低运维成本 X%
- **质量**：缺陷率降低 X%、可用性提升至 X%
- **规模**：服务 X 万用户、支撑 X QPS
- **增长**：收入增长 X%、转化率提升 X%
- **速度**：部署时间从 X 缩短至 Y、响应时间降低 X%

**若用户未提供具体数据**：
1. 主动追问："这项工作大约提升了多少效率？节省了多少成本？影响了多少用户？"
2. 若用户确实无法提供，使用范围量化（"管理 10+ 人团队"、"服务 3 个业务线"）替代

### 阶段三：生成简历

使用 assets/resume_template.html 模板生成简历文件。

**布局规范**（2025-2026 现代简历设计趋势）：
- **双栏侧边栏布局**：左侧窄栏放联系信息 + 技能 + 教育；右侧主栏放摘要 + 工作经历 + 项目经历
- **配色方案**：主色使用专业蓝灰色系（#2c3e50 侧边栏背景 + #3498db 强调色），正文使用 #333 深灰
- **字体**：中文使用思源黑体或系统默认无衬线字体，英文使用 Inter / Helvetica Neue
- **视觉层次**：姓名 24-28px 加粗、板块标题 14-16px 大写 + 下划线分隔、正文 12-13px
- **留白**：页边距 0.5-0.7inch，各板块间距 12-16px
- **单页原则**：除非高级管理岗，严格控制在 1 页以内

**生成步骤**：
1. 读取 assets/resume_template.html 模板
2. 将收集和优化后的内容填入对应模块
3. 写出最终 resume.html 到用户工作目录
4. 使用浏览器预览呈现给用户

### 阶段四：迭代优化

生成后主动询问：
1. 是否需要调整某段经历的 STAR 描述？
2. 是否需要增删板块？
3. 是否需要调整配色风格？
4. 是否需要导出为 PDF？（提示使用浏览器打印功能 Ctrl/Cmd + P 导出）

## STAR 法则快速参考

### 强力动词库

| 类别 | 动词 |
|------|------|
| 领导力 | 主导、统筹、推动、带领、牵头、协调 |
| 技术 | 设计、架构、开发、重构、优化、搭建、实现 |
| 分析 | 分析、诊断、评估、调研、建模、预测 |
| 创新 | 首创、引入、探索、创建、革新、开拓 |
| 管理 | 管理、运营、维护、监控、治理、把控 |
| 提升 | 提升、改进、缩短、降低、消除、加速 |

### 量化模板

```
效率类: "将[指标]从[旧值]优化至[新值]，提升X%"
成本类: "年度节省[金额]运营成本 / 降低X%的[资源]消耗"
规模类: "支撑[X]万日活用户 / [X]QPS并发请求"
质量类: "系统可用性从[X]%提升至[Y]% / 缺陷率降低X%"
增长类: "带动[指标]增长X% / 新增[X]个客户"
速度类: "将[流程]耗时从[X]缩短至[Y]，效率提升X%"
```
