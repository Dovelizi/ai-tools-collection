### 步骤 2: 查询日志

**执行要求**: 必须调用 `cls-mcp-server` 的 `SearchLog` 获取日志

#### 日志查询参数 (JSON 格式)

```json
{
  "Region": "ap-nanjing-xx",
  "TopicId": "xxxx",
  "Query": "<根据下方规则构建>",
  "From": <Unix时间戳毫秒>,
  "To": <Unix时间戳毫秒>,
  "Limit": 5,
  "Sort": "asc"
}
```

#### Query 语句构建规则

**基础模板**:

```
'reportmonitorcontext'
```

**条件追加规则** (根据用户输入动态添加):


| 用户输入           | Query 追加内容             | 示例                                         |
| ------------------ | -------------------------- | -------------------------------------------- |
| 方法: xxx          | `and 'xxx'`                | `and 'blameAmount'`                          |
| 业务码: xxx        | `and 'xxx'` (除非是"其他") | `and 'sp_60708'`                             |
| 错误码: xxx        | `and 'xxx'`                | `and 'payType为0但是有需要退款的账单:60723'` |
| 包含"上升"或"下降" | 仅使用方法名               | `"Query": "blameAmount"`                     |

**成功标志处理**:

- 首次查询: 添加 `and 'success=false'`
- 若无结果: 移除该条件重新查询

**完整示例**:

```
'blameAmount' and 'reportmonitorcontext' and 'success=false' and 'sp_60708' and 'payType为0但是有需要退款的账单:60723'
```

#### 时间范围计算规则

**情况 A: 用户提供时间**

用户输入示例: `发生时间: 2025-11-12 15:50:00`，时间格式为`yyyy-MM-dd HH:mm:ss`，如果读取到的格式不对需要重新读取。

**计算步骤** (必须严格执行):

1. **获取当前时间基准**

   ```
   必须先调用 GetCurrentTimestamp 获取当前时间基准
   ```
2. **计算目标时间戳**

   ```
   解析用户时间: 2025-11-12 15:50:00
   从 Unix 纪元 (1970-01-01)开始计算，转换为 Unix 时间戳(秒)
   乘以 1000 转换为毫秒
   ```
3. **反向验证** (防止错误)

   ```
   必须将计算出的时间戳与GetCurrentTimestamp基准时间做比对
   必须逐项对比: 年、月、日、时、分、秒，差值不允许超过30天
   ```
4. **计算查询窗口**

   ```
   From = 目标时间戳 - 300000  (向前推 5 分钟)
   To   = 目标时间戳 + 300000  (向后推 5 分钟)
   ```

**示例计算**:

```
2025-11-11 00:00:00 的时间戳:
- 从 Unix 纪元 (1970-01-01) 到 2025-11-11
- 这是一个固定值，需要精确计算

2025-11-11 00:00:00 ≈ 1762876800 秒 = 1762876800000 毫秒
加上 20:47:00 = 20 * 3600 + 47 * 60 = 72000 + 2820 = 74820 秒

告警时间戳 = 1762876800000 + 74820000 = 1762951620000 毫秒
From = 1762951620000 - 300000  (向前推 5 分钟)
To   = 1762951620000 + 300000  (向后推 5 分钟)
```

**情况 B: 用户未提供时间**

```
当前时间戳(毫秒): current_timestamp
From: current_timestamp - 900000  (最近 15 分钟)
To:   current_timestamp
```

## 时间戳计算辅助工具

**Unix 时间戳参考** (方便手动验证):

```
2025-11-12 00:00:00 = 1762876800 秒 = 1762876800000 毫秒
1 分钟 = 60000 毫秒
5 分钟 = 300000 毫秒
15 分钟 = 900000 毫秒
```

**验证方法**:

```python
# Python 示例
from datetime import datetime
timestamp_ms = 1762933800000
dt = datetime.fromtimestamp(timestamp_ms / 1000)
print(dt)  # 应输出: 2025-11-12 15:50:00
