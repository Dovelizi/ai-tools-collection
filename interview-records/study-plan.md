## 🔴 高优先级（P0）— 面试前必须掌握

### 1. 虚拟线程（Virtual Threads）原理

- **所属领域**：Java 基础 / 并发
- **当前状态**：🆕 新增（本次面试暴露）
- **薄弱程度**：⚠️ 方向对但表述不精确
- **暴露次数**：1 次（2026-05-13）
- **具体缺失点**：
  - [2026-05-13] 认为虚拟线程"由用户控制"，实际是 JVM 调度到 carrier 线程
  - [2026-05-13] 未提到 M:N 模型（M 个虚拟线程跑在 N 个平台线程上）
  - [2026-05-13] 未提到 pinned 问题（synchronized 块中虚拟线程无法让出）
  - [2026-05-13] 未提到不要池化、ThreadLocal 慎用
- **学习目标**：从"方向对不精确"提升到"能精准讲清原理和三大注意点"
- **学习方法**：
  1. 理解 M:N 调度模型（虚拟线程 → carrier 线程池 → OS 线程）
  2. 背三个面试核心坑：不要池化、synchronized 导致 pinned、ThreadLocal 慎用
  3. 理解为什么 IO 密集型场景优势最明显（yield 机制）
  4. 知道虚拟线程不是"用户直接控制"而是 JVM 管理的
- **预计时长**：2 小时
- **验证方式**：能说清 M:N 模型 + 三大注意点

### 2. MySQL MVCC 结构化表述

- **所属领域**：中间件 / MySQL
- **当前状态**：🆕 新增（本次面试暴露）
- **薄弱程度**：⚠️ 方向正确但表述不够结构化
- **暴露次数**：1 次（2026-05-13）
- **具体缺失点**：
  - [2026-05-13] 知道三大组件（隐藏字段+undo log+ReadView）但表述散乱
  - [2026-05-13] ReadView 判断规则不完整（creator_trx_id / min_trx_id / max_trx_id / m_ids）
  - [2026-05-13] RR vs RC 差异能说清方向但不够结构化
- **学习目标**：从"方向对不结构化"提升到"能流畅讲出完整判断链"
- **学习方法**：
  1. 先讲三大组件：隐藏字段(DB_TRX_ID/DB_ROLL_PTR) + undo log 版本链 + ReadView
  2. 背诵 ReadView 五条判断规则
  3. 对比 RR（一次 ReadView 复用）vs RC（每次 SELECT 新 ReadView）
  4. 记忆口诀：RR 是"拍一次照片看全程"，RC 是"每次拍照看当前"
- **预计时长**：1.5 小时
- **验证方式**：能完整说出三大组件 + ReadView 五条判断规则 + RR/RC 差异
